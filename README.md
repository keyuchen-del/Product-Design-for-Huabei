# 🌸 花小呗情感化设计方案

> 从「工具」到「伙伴」的设计重构 —— 让花小呗不只是 AI 助手，而是你的 AI 伙伴

## 项目概述

这是一份面向花呗 AI 助手「花小呗」的情感化设计方案，包含完整的设计文档与可交互的前端 Demo。

核心命题：**如何把花呗生态里的冰冷数据，转化为一段温暖而有边界的 AI 关系。**

## 在线 Demo

👉 [点击体验](https://keyuchen-del.github.io/huaxiaobei-demo/)

Demo 包含四个核心页面：

| 页面 | 说明 |
|------|------|
| 智能对话 | Day 0 见面礼交互流程，含建议卡片、加载状态、反馈组件 |
| 亲密度 | 亲密度权益体系，对齐花呗现有设计语言 |
| 小本本 | 三层记忆架构可视化（先验层 / 共建层 / 默契层） |
| 拒额五幕剧 | 拒额场景的五幕式情感化交互 Demo |

## 设计亮点

**记忆体系**：提出先验层、共建层、默契层三层记忆架构，解决「开箱即懂」与「被监控感」之间的设计张力。其中「禁区锚点」和「未完待续锚点」是市面上尚无 AI 产品真正做好的两个维度。

**拒额五幕剧**：将行业最冷漠的拒额场景重构为五幕式情感交互（通知 → 共情 → 解释 → 行动 → 闭环），72 小时后主动回访是整个设计最关键的一笔。

**三序取舍原则**：用户自主 > 系统建议 > 商业目标。任何一次花小呗的发言违反此顺序，都视为设计失败。

**情绪系统**：依托花小呗的 Q 版吉祥物形态，通过动作 + 表情 + 文字三模态表达情绪，构建其他纯文字 AI 无法复制的差异化护城河。

## 技术说明

- 前端 Demo 为纯 HTML/CSS/JS 单文件，无外部依赖
- 设计语言严格对齐支付宝 / 花呗现有产品 UI
- 适配移动端 390px 视口，建议用手机浏览器或 Chrome DevTools 手机模式查看
